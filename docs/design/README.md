# Design Documentation

This directory contains design specifications, briefs, and system definitions for the Handover Player.

## Files

- **`DESIGN_SYSTEM_EXPORT.md`**: (Active) The current Single Source of Truth for the Design System v2. Contains tokens, component specs, and layout rules.
- **`DESIGN_BRIEF.md`**: Initial design requirements and conceptual outline.
- **`DESIGN_GUIDELINES.md`**: General design guidelines.
- **`DESIGN_FROM_PENCIL.md`** / **`DESIGN_ORDER_PENCIL.md`**: Raw design outputs/orders likely from the prototyping phase.
- **`HANDOVER_REQUEST.md`**: Documentation regarding the handover process/requirements.

## Current Status (v2.0.0)
The design system has been implemented in the codebase as of v2.0.0.
Refer to `src/index.css` and `tailwind.config.js` for the implementation code.
